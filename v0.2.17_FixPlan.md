## Bug 1: 研发项目导入后 method_type 错误 🔴

**文件**: `workload-tool-rust\v0.2.17\src\repo\method_repo.rs`  
**位置**: 第 230 行  
**错误**: INSERT 没有指定 method_type，默认值='其他'，导致导入的项目不显示在"研发项目管理"卡片

**当前代码**:
```rust
conn.execute(
    "INSERT INTO projects (group_id, name) VALUES (?1,?2)",
    rusqlite::params![proj_gid, pname],
)?;
```

**改为**:
```rust
conn.execute(
    "INSERT INTO projects (group_id, name, method_type) VALUES (?1,?2,'研发项目')",
    rusqlite::params![proj_gid, pname],
)?;
```

## Bug 5: 方法导入时审计统计计数不对 🟡

**文件**: `workload-tool-rust\v0.2.17\src\repo\method_repo.rs`  
**位置**: 第 270-281 行  
**错误**: `type_counter` 统计的是每条 method_type 出现的次数（含已存在方法），但 `method_count` 只计数新插入的。审计日志中的数字和实际导入预览不一致。

**不需要改代码**，这是因为 type_counter 统计的是"检测到的行数"（含跳过），而 method_count 统计的是"实际新增数"。这就是想要的语义。  
**但是如果导入预览显示"新增0条"但数据实际存在，请联系 Bug1 修复。**

## Bug 6: 回收站数据查询 🟡

**文件**: `workload-tool-rust\v0.2.17\src\api\record_handler.rs`  
**错误**: `GET /api/records` 的 list 函数可能过滤了 `deleted_at IS NULL` 的记录，导致回收站看不到已删除记录

**检查**: 在 `src\repo\record_repo.rs` 中确认 list 查询是否有 `AND deleted_at IS NULL` 过滤条件。如果有，需要增加一个 `include_deleted` 参数或新增 `/api/records/deleted` 端点。

**前端已修复** (`ManagePage.tsx` L67): `filter(x => x.deleted_at != null)` ✅
