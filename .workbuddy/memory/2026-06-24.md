## Rust 工作量统计工具 — 路由 405/HTML Bug 根因修复

### 问题
- 所有带路径参数的 API 路由（`PUT /api/records/1`、`DELETE /api/records/1` 等）返回 HTML 而非 JSON
- 静态路由（`GET /api/records`、`GET /api/groups`）正常返回 JSON
- 请求落入 SPA fallback `serve_index`，返回 `index.html`

### 根因
**axum 0.7.9（matchit 0.7.3）使用 `:id` 语法，不是 `{id}` 语法**

- `{id}` 语法是 axum 0.8（matchit 0.8）才引入的
- 在 matchit 0.7.3 中，`{id}` 被当成字面量字符串，路由 `/api/records/{id}` 只匹配 URL `/api/records/{id}` 本身
- 所有带数字 ID 的请求（如 `/api/records/1`）无法匹配，落入 fallback

### 修复内容
1. **路径参数语法**：所有 handler 中 `{id}` → `:id`，`{user_name}` → `:user_name`
2. **路由架构**：`.nest()` → `.merge()` + 完整路径前缀（每个 handler 使用 `/api/xxx/:id` 而非相对路径）
3. **serve_index 编译错误**：`(String, StatusCode)` → `(StatusCode, String)` 修复 axum 0.7 IntoResponse trait bound
4. **SPA fallback 恢复**：`.fallback(serve_index)` 正确配置，读取 `static/index.html`

## 4 个致命 Bug 修复 (BugFix 2026-06-24 下午)
### Bug 1 (P0): export_handler 路由缺少 `/api` 前缀
- 在 `.nest()` → `.merge()` 重构时遗漏，路由为 `/export/excel` 而非 `/api/export/excel`
- 前端请求落入 SPA fallback，返回 HTML 而非 Excel → 表现为"导出模板错误"+"文件损坏"
- 修复：`/export/excel` → `/api/export/excel`

### Bug 2 (P1): by_instrument SQL 运算符优先级错误
- `WHERE p.name LIKE '%LC-%' OR p.name LIKE '%GC-%' AND {wc}` 等价于 `WHERE ... OR (... AND {wc})`
- LC 项目不受日期过滤，导致"按仪器类型分类错误"
- 修复：添加括号 `(LIKE '%LC-%' OR LIKE '%GC-%')`

### Bug 3 (P1): InstrumentStats 缺少 instrument_type 字段
- 前端 TypeScript 类型有此字段，后端 struct 和 SQL 不返回
- 修复：struct 新增字段 + SQL 新增 CASE WHEN 计算

### Bug 4 (P2): export_handler month_bounds 日期边界
- `d_end` 为下月第一天 + `format!("{}T23:59:59", d_end)` + `<=` 会包含下月首日数据
- 修复：`<= ?2` → `< ?2`，去掉 T23:59:59 后缀
- 影响位置：`src/api/export_handler.rs` 第 140/144/314/322 行

- `src/api/export_handler.rs` — Bug 1 + Bug 4（路由前缀 + 日期边界）
- `src/api/stats_handler.rs` — Bug 2 + Bug 3（SQL 括号 + instrument_type）
