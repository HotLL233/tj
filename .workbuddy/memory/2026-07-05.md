# 2026-07-05 工作日志

## v0.3.15 发布
- **Bug 修复 1**：`record_repo.rs` 的 `get_by_id_on_conn()` 函数仍使用 `LIMIT 1` 子查询获取实验室名（v0.3.14 漏改了单条记录查询），改为 `LEFT JOIN project_lab_links`
- **Bug 修复 2**：`ManagePage.tsx` 初始 `useEffect` 没有调用 `lmt()`（加载方法类型），导致项目编辑对话框中"关联检测方法"按类型分组展示时 `mts` 为空，方法列表不显示。需先点"方法管理"卡片触发加载才能显示。修复：在初始 `Promise.all` 中加入 `lmt()`

## v0.3.16 发布
- **Bug 修复（关键）**：v0.3.14/0.3.15 将 `LIMIT 1` 子查询改为 `LEFT JOIN project_lab_links + LEFT JOIN project_groups` 后，一个关联了 N 个实验室的项目会把同一条记录展开成 N 行（用户录入 1 条显示多条）
- **修复方案**：将 `JOIN` 方式改为 `group_concat` 标量子查询——实验室名用逗号拼接显示，确保一条记录只返回一行
- **修改文件**：仅 `record_repo.rs` 的 `list()` 和 `get_by_id_on_conn()` 两个函数
- **教训**：用 JOIN 替代 LIMIT 1 时必须考虑一对多关系会导致行膨胀，聚合场景应用子查询而非 JOIN

## v0.3.17 发布
- **Bug 修复（关键）**：`export_data.rs` 的 Sheets 1/2/3/4/5/7 查询函数同样存在 `LEFT JOIN project_lab_links` 导致导出数量翻倍的问题（与 v0.3.15 相同的根因）
- **修复方案**：所有受影响的查询函数统一改为 `group_concat` 标量子查询获取实验室名，移除 `LEFT JOIN project_lab_links/project_groups`
- **格式确认**：`export_write.rs` 的 `Fmt` 结构体已正确设置仿宋 14 号、水平垂直居中，无需改动
- **修改文件**：`export_data.rs`（6 个查询函数）、`Cargo.toml`、`Layout.tsx`

## v0.3.19 发布
- **Bug 修复 1（导出翻倍）**：v0.3.18 的 LEFT JOIN project_lab_links 导致所有汇总表（Sheets 1-4,7）行膨胀，SUM(quantity) 翻倍。修复方案：**子查询先聚合 work_records（按 project_id + method_id），再 LEFT JOIN 实验室**。内层完成聚合后外层 JOIN 只展开显示维度，不影响 quantity
- **Bug 修复 2（人员汇总公式丢失）**：`export_write.rs` 的 `write_sheet6` 中 merge K 列时传入空字符串 `""`，覆盖了之前写入的个人汇总公式。修复：merge 时改用 `=SUM(CK_start:CK_end)` 公式
- **需求 3（方法一览直接内联编辑）**：去掉操作列和双击切换模式。可编辑列（方法名称、系数、单价）直接渲染为 TextField，失焦自动调用 updateMethod API 保存；类型用 Chip 点击切换
- **修改文件**：`export_data.rs`（6 个查询函数改为子查询模式 + Sheet 5 改回 group_concat）、`export_write.rs`（Sheet 6 K 列 merge 公式）、`ManagePage.tsx`（方法一览重写为内联编辑）、`Layout.tsx`
- **SQL 模式对比（v0.3.19 最终方案）**：
  - 汇总表（Sheets 1-4, 7）：`FROM (聚合子查询) agg LEFT JOIN project_lab_links ...` — 先聚合再 JOIN
  - 原始记录表（Sheet 5）：`group_concat` 子查询 — 保持一行一记录
