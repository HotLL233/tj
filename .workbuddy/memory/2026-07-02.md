## v0.3.5 — EntryPage "今日记录"不显示修复

### 根因
`static/index.html` 引用的是旧版 JS `index-BwYY9H9t.js`（13:40 构建，不含"今日记录"代码），而后端 serve 的是 `static/` 目录。`dist/` 下的 `index.html` 引用新版 `index-B4sArT9N.js` 但 runInBackground 构建未同步到 `static/`。

### 修复
1. 前端重新构建: `cd project-root/frontend && npx vite build` → 输出到 `../backend/static/`
2. 复制新构建到 `v0.3.5/static/`: index.html + assets/index-B4sArT9N.js（含"今日记录"）
3. 编译 Rust 二进制: `cargo build --release`（前台执行）
4. 打包安装程序: Inno Setup 6 ISCC.exe → `installer/工作量统计工具_Rust_v0.3.5_Setup.exe`

### 教训
- 用户偏好：所有操作必须在**前台执行**，不使用 run_in_background
- 每次前端构建后必须确保 `static/` 被更新（不是 dist/）

## v0.3.6 BugFix — 导出 Excel 报错

### 问题
用户安装 v0.3.6 后导出的 Excel 文件打开报错。排查发现所有 10 个 Sheet 返回 0 行数据。

### 根因（两个）
1. **安装包覆盖空数据库**：`build_installer.iss` `dist\*` recursesubdirs 包含 `dist/data/workload.db`（新 schema 但 0 行空库），安装时覆盖用户数据库
2. **Sheet 1 SQL 查询链错误**：`export_data.rs` L170-171 使用 `JOIN project_lab_links pll ON pg.id = pll.group_id`，该表仅迁移了"研发项目"/"实验室管理"类型数据，其他项目无记录

### 修复
1. `build_installer.iss` L24-26：改为明确列出 exe + static/* + icon.ico，排除 data/ 目录
2. `export_data.rs` L170-171：改为 `JOIN projects p ON p.group_id = pg.id`

### 编译打包
- 编译成功（1 warning: unused import HashMap）
- Inno Setup 生成：`installer/工作量统计工具_Rust_v0.3.6_Setup.exe`（5.2MB）
- 验证：安装包不再包含 data/ 目录（仅 workload-tool.exe + static/* + icon.ico）
