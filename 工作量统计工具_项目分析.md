### 数据Sheet格式

```
| 日期     | 个数     | 合计          |
|----------|----------|---------------|
| 20260617 | 5        | =SUM(B:B)     |
|          | 6        |               |  ← 空日期=同一天
|          | 5        |               |
| 20260618 | 3        |               |
```

- 日期列：YYYYMMDD 格式（8位无分隔符），空单元格=同一日期延续
- 个数列：该批次测试的样品数量
- 合计列：全列 SUM 公式

## 三、与现有检测数据统计工具的区别

| 维度 | 检测数据统计工具 | 工作量统计工具 |
|------|----------------|--------------|
| 输入列 | 日期 + **批号** | 日期 + **个数** |
| 统计方式 | `nunique()` 去重计数 | `sum()` 求和 |
| 输出列 | 每个Sheet一种批号统计 | 每个方法工作量的每日汇总 |
| 日期格式 | YYYY-MM-DD | **YYYYMMDD**（8位无分隔） |
| 跳过Sheet | 跳过"目录" | 跳过"目录"+"模板"+"各实验室项目方法名" |
| 合计行 | 各列合计 | 各方法求和 + 总合计 |

## 四、技术方案

### 复用现有基础设施（≈90% 代码复用）

| 模块 | 来源 | 改动 |

### 仅需修改的 3 个核心函数

1. **日期解析** — `ensure_date_str()` 已支持 YYYYMMDD ✅ （已有 `try_parse_yyyymmdd`）
2. **统计算法** — `process_sheet()` 中 `HashSet→nunique` 改为直接 `sum`
3. **Sheet 跳过列表** — 增加 "目录" / "模板" / "各实验室项目方法名"

### 统计算法变更（核心）

```rust
// 原版（检测数据统计）：去重计数
let mut daily: BTreeMap<String, HashSet<String>> = BTreeMap::new();
for (d, b) in &recs { daily.entry(d.clone()).or_default().insert(b.clone()); }
let count = daily[date].len();  // HashSet.len() = unique count

// 新版（工作量统计）：直接求和
let mut daily: BTreeMap<String, usize> = BTreeMap::new();
for (d, count_str) in &recs {
    let c: usize = count_str.parse().unwrap_or(0);
    *daily.entry(d.clone()).or_insert(0) += c;  // 累加个数
}
let sum = daily[date];  // 直接求和的合计
```

## 六、关键踩坑提醒（从检测工具项目继承）

1. **必须用 `cargo tauri build`**，不能用 `cargo build --release`（不嵌入前端，白屏）
2. **数值写入必须用 `write_number_with_format`**，不能用 `write_string_with_format`（Excel 当文本，SUM 失效）
3. **SUM 公式范围**：`data_end = all_dates.len() + 1`（Excel 行号从 1 开始，数据从行 2 开始）
4. **空单元格也要写 "" + 边框格式**（保证全表格外框线）
5. **自动列宽**：`ch_width(汉字=2, ASCII=1) + 5`，`clamp(8, 30)`
6. **Tauri IPC 参数映射**：前端 `camelCase` → 后端 `snake_case` 自动转换
7. **Inno Setup**：`PrivilegesRequired=lowest`，装到 `{localappdata}`
