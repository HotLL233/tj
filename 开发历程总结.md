## 一、项目概述

### 1.1 项目定位

**工作量统计工具**是一个桌面级 Web 应用，用于实验室环境下多用户工作量录入、统计分析、数据导出和审计追溯。核心用户场景：实验室人员按分组 → 项目录入每日样品检测数量，管理人员按周/月/用户/项目/仪器类型/仪器编号等维度统计汇总并导出 Excel。

### 1.2 演变历程

| 阶段 | 版本 | 技术栈 | 关键变化 |
|------|------|--------|---------|
| 原型 | v1.0.0 ~ v1.7.2 | Python (FastAPI) + React | Excel 解析/多维度统计/系统托盘/卡片 UI |
| Rust 重写 | v0.1.0 → v0.1.2 | Rust (Axum 0.7) + React (存量) | 后端重写为 Rust，修复路由 Bug，保持 API 兼容 |

项目最初是一个基于 Tauri (Rust) 的桌面工具（2026-06-19），随后转向 FastAPI + React Web 应用（2026-06-21），最终后端重写为纯 Rust (Axum 0.7.9)，保留前端不变。

### 4.1 数据模型 (DDL)

```sql
-- 4 张表，WAL 模式，外键约束开启
project_groups (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL UNIQUE,          -- 实验室名称 (410/415/417/418...)
    sort_order INTEGER DEFAULT 0,      -- 排序权重

projects (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    group_id INTEGER REFERENCES project_groups(id),
    name TEXT NOT NULL,                 -- 项目代号 (HYLY-LC-01(230106)...)

work_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id INTEGER REFERENCES projects(id),
    user_name TEXT NOT NULL,            -- 录入人
    quantity INTEGER CHECK(quantity > 0), -- 样品数量
    recorded_at TEXT NOT NULL,           -- 录入日期时间
    created_at TEXT,
    deleted_at TEXT                     -- NULL=正常, 非NULL=已删除 (软删除)
)
-- 索引: (project_id), (recorded_at), (user_name), (deleted_at)

audit_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    action TEXT NOT NULL,               -- create/update/delete/restore
    table_name TEXT NOT NULL,           -- work_records/project_groups/projects
    record_id INTEGER,
    user_name TEXT,
    detail TEXT,
    created_at TEXT
)
```

### 4.3 事务原子性设计

所有写操作通过 `conn.transaction()` 包裹，审计日志通过 `log_on_conn(&tx, ...)` 写入同一事务：

```rust
pub fn update(pool: &DbPool, id: i64, body: &RecordUpdate, user_name: &str) -> Result<RecordResponse> {
    let mut conn = pool.get()?;
    let tx = conn.transaction()?;                         // 1. 开启事务
    let existing = get_by_id_on_conn(&tx, id)?;            // 2. 事务内查询
    if existing.deleted_at.is_some() {
        return Err(AppError::Validation("记录已被删除".into()));
    }
    // 3. UPDATE + 影响行数检查
    let rows = tx.execute("UPDATE work_records SET ... WHERE id=?", [...])?;
    if rows == 0 { return Err(AppError::NotFound("记录不存在".into())); }
    audit_repo::log_on_conn(&tx, "update", ...)?;          // 4. 同事务审计
    tx.commit()?;                                           // 5. 提交事务
    get_by_id_on_conn(&conn, id)                            // 6. 返回最新数据
}
```

### 4.4 统计维度实现

#### 按类型分类（液相/气相）
```sql
-- 基于项目名称中的 GC-/LC- 前缀自动分类
CASE WHEN p.name LIKE '%GC-%' THEN '气相'
     WHEN p.name LIKE '%LC-%' THEN '液相'
     ELSE '其他'
END AS instrument_type
```

#### 按仪器分类
```sql
-- 从项目名称中提取仪器编号（如 HYLY-LC-01）
SUBSTR(p.name, INSTR(p.name,'-')+1) AS instrument
```

#### 按时间粒度（天/周/月）
```rust
let period_expr = match group_by {
    "week"  => "strftime('%Y-W%W', wr.recorded_at)",   // 年的第几周
    "month" => "strftime('%Y-%m', wr.recorded_at)",     // 年月
    _       => "date(wr.recorded_at)",                   // 日期（默认）
};
```

### 4.5 路由设计 (CRITICAL 教训)

**axum 0.7.9 (matchit 0.7.3) 使用 `:id` 语法，非 `{id}`！**

| 语法 | 版本 | 行为 |
|------|------|------|
| `:id` | axum 0.7 (matchit 0.7) | ✅ 参数捕获，匹配任意单段 |
| `{id}` | axum 0.8 (matchit 0.8) | ❌ axum 0.7 中被当作字面量 |

采用 `.merge()` + 完整路径前缀架构：

```rust
// 每个 handler 使用完整路径
pub fn router(pool: DbPool) -> Router {
    Router::new()
        .route("/api/records", get(list).post(create))
        .route("/api/records/:id", put(update).delete(soft_delete))
        .route("/api/records/restore/:id", post(restore))
        .route("/api/records/by-user/:user_name", delete(delete_by_user))
        .with_state(pool)
}

// mod.rs 用 merge 组装
pub fn api_router(pool: DbPool) -> Router {
    Router::new()
        .route("/api/version", get(version))
        .merge(group_handler::router(pool.clone()))
        .merge(project_handler::router(pool.clone()))
        .merge(record_handler::router(pool.clone()))
        // ...
}
```

### 4.6 SPA Fallback 机制

```rust
let app = api::api_router(pool)
    .nest_service("/assets", ServeDir::new("static/assets"))  // 静态资源（JS/CSS/图片）
    .fallback(serve_index)                                     // SPA fallback
    .layer(CorsLayer::permissive());                           // CORS 全开

async fn serve_index() -> impl IntoResponse {
    match tokio::fs::read_to_string("static/index.html").await {
        Ok(html) => ([(CONTENT_TYPE, "text/html")], html).into_response(),
        Err(_) => (StatusCode::NOT_FOUND, "index.html not found").into_response(),
    }
}
```

### 4.7 错误处理统一策略

```rust
#[derive(Debug, thiserror::Error)]
pub enum AppError {
    Database(#[from] rusqlite::Error),    // 数据库错误 → code:500
    Pool(#[from] r2d2::Error),            // 连接池错误 → code:500
    Xlsx(#[from] rust_xlsxwriter::Error), // Excel 错误 → code:500
    NotFound(String),                     // 未找到 → code:1
    Validation(String),                   // 验证失败 → code:1
    Conflict(String),                     // 冲突 → code:1
    Internal(String),                     // 内部错误 → code:500
}
// 所有错误统一返回 HTTP 200 + 业务错误码
```

## 七、已知问题与修复记录

| 日期 | 问题 | 根因 | 修复 |
|------|------|------|------|
| 2026-06-24 | 所有路径参数路由返回 HTML | `{id}` 语法不兼容 axum 0.7 (matchit 0.7.3) | 改为 `:id` 语法 |
| 2026-06-24 | serve_index 编译错误 | `(String, StatusCode)` tuple 顺序错误 | 改为 `(StatusCode, String)` |
| 2026-06-24 | record_repo 事务原子性缺陷 | 审计日志跨连接，缺少事务包裹 | `conn.transaction()` + `log_on_conn(&tx, ...)` |

### 7.1 axum 0.7 → 0.8 迁移注意事项

如果未来升级到 axum 0.8，需要修改：
1. `:id` → `{id}`（路径参数语法变更）
2. matchit 0.8 的行为差异（路由冲突处理更严格）
3. `.merge()` 在 0.8 中的 API 签名可能变化

// baseURL: '/api', timeout: 10000ms, Content-Type: application/json

// Groups

createRecord(data)       → POST   /records
updateRecord(id, data)   → PUT    /records/${id}
deleteRecord(id)         → DELETE /records/${id}
restoreRecord(id)        → POST   /records/restore/${id}
deleteRecordsByUser(name)→ DELETE /records/by-user/${encodeURI(name)}?start&end

// Stats
getStatsSummary(params)  → GET    /stats/summary?start&end&group_by&group_id
getStatsByUser(params)   → GET    /stats/by-user?start&end
getStatsByProject(params)→ GET    /stats/by-project?start&end&group_id
getStatsByType(params)   → GET    /stats/by-type?start&end
getStatsByInstrument(p)  → GET    /stats/by-instrument?start&end
