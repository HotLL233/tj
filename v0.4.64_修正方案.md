### 实施方法（**仅后端，无新依赖**）

**模板生成**：
- 浏览器端用 `Blob` + `URL.createObjectURL` 生成 CSV
- 模板内容：
  ```
  用户名,密码,真实姓名,所属部门,所属实验室,角色,启用
  zhangsan,123456,张三,研究部,410,普通用户,1
  ```

**关键代码**（前端）：
```tsx
// 1. 定义模板内容常量
const USER_TEMPLATE = "用户名,密码,真实姓名,所属部门,所属实验室,角色,启用\nzhangsan,123456,张三,研究部,410,普通用户,1\n";

// 2. 下载按钮
<Button onClick={() => {
  const blob = new Blob(["\uFEFF" + USER_TEMPLATE], { type: 'text/csv;charset=utf-8;' });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url; a.download = 'users_template.csv';
  a.click(); URL.revokeObjectURL(url);
}}>下载模板</Button>
```

**注意**：`\uFEFF` 是 BOM 头，确保 Excel 打开时正确识别 UTF-8 中文。

### 改动文件
- `frontend/src/pages/ManagePage.tsx` — 在批量导入对话框中加"下载模板"按钮（在"选择文件"前）+ USER_TEMPLATE 常量

## 2. 修正 2：高项改为"自定义填写"

### 需求分析

用户原话："关联高项不是说的自定义填写吗" + "自定义模式" + "类似于项目，和项目平级"

**正确理解**：
- "自定义填写"= 用户在项目编辑框里**直接输入**高项名称（不需要先在另一处创建）
- "和项目平级"= 高项**没有自己独立的 CRUD 页面**（不像"项目"那样有"新建项目"按钮）
- "类似项目"= 项目本身有"项目名"字段（直接输入），高项类比 = 在项目编辑里直接输入高项名

**错误实现**：我把"高项"做成了独立表 + 独立 CRUD 页面 + Select 下拉，违反了"自定义填写"。

### 修正方案

**两种实现**：

#### 方案 A：纯文本字段（最简单）
- `projects` 表加 `high_item` 字段（TEXT 字符串）
- 项目编辑对话框加 `TextField` 输入框
- 录入页不显示
- 统计 sheet 加"高项"列

#### 方案 B：自动补全文本字段（推荐）
- `projects` 表加 `high_item` 字段（TEXT 字符串）
- 项目编辑对话框加 `Autocomplete` freeSolo 输入框
- 用户可输入新值，也可从历史高项里选
- 录入页不显示
- 统计 sheet 加"高项"列

**推荐方案 B**。代码量略多但用户体验最好。

### 数据库调整

```sql
-- 移除 v0.4.63 的 high_items 表（外键约束会要求清理）
-- 或保留作为历史值的 dedup 表，但 projects 用 TEXT 字段

ALTER TABLE projects ADD COLUMN high_item TEXT DEFAULT NULL;
-- 保留 v0.4.63 的 high_items 表作为参考表（可选）
-- 但 projects.high_item_id 改为 projects.high_item TEXT
```

### 前端 UI（项目编辑对话框）

```tsx
{/* v0.4.64: 关联高项 — 自定义填写 */}
<Autocomplete
  freeSolo
  options={Array.from(new Set(projects.map(p => p.high_item).filter(Boolean)))}
  value={projectEditItem.high_item || null}
  onChange={(_, v) => setProjectEditItem({ ...projectEditItem, high_item: v || '' })}
  onInputChange={(_, v) => setProjectEditItem({ ...projectEditItem, high_item: v })}
  renderInput={(params) => (
    <TextField {...params} label="关联高项（自定义填写）" placeholder="例如：高项/A 类/重点" size="small" sx={{ mt: 2 }} />
  )}
/>
```

### 改动文件

## 3. 修正 3：检测人记录页加"高项"列

### 现状

**截图显示的页面**：检测人记录（按"检测人"筛选的列表），列：
- 序号
- 日期时间
- 检测人
- 实验室
- 项目
- 方法
- 类型
- 仪器
- 数量
- 单价倍率
- （缺：高项）

**位置**：`EntryPage.tsx` 的"今日记录"或"全部记录" 表格，**或** `StatsPage.tsx` 中"检测人记录"模块。

### 关键发现

检查代码后：
- `EntryPage.tsx:240-250` 今日记录表格**没有"高项"列**
- `SampleEntryPage.tsx` 同样没有
- `StatsPage.tsx` 同样没有

### 修正方案

**记录列表加 1 列"高项"**：
- 列名：`高项`
- 显示：`rec.high_item || '-'`
- 数据来源：后端 `work_records` 表加 `high_item` 字段（从 `projects.high_item` 关联时复制过来，避免每条记录都 JOIN）

**为什么 records 要存高项**：
- 工作记录 = 用户当时录的数据快照
- 高项是项目属性，如果改项目的"高项"，历史记录不应该变
- 所以 `work_records.high_item` 在 create 时从 `projects.high_item` 复制

**统计 sheet 加"高项"列**：v0.4.63 方案里有 ABC 三方案全要，但 v0.4.63 没做。v0.4.64 一起补上。

### 数据库调整

```sql
-- work_records 加 high_item 字段（创建时从项目复制）
ALTER TABLE work_records ADD COLUMN high_item TEXT DEFAULT NULL;
-- 研发送样表同样加
ALTER TABLE rd_work_records ADD COLUMN high_item TEXT DEFAULT NULL;
```

### 后端改动

- `src/api/record_handler.rs` — `create_record` 时从 projects 表查 high_item 复制
- `src/api/rd_record_handler.rs` — 同样
- `src/api/stats_handler.rs` — 统计 SQL 加 high_item 列
- `src/api/export_data.rs` / `export_write.rs` — 所有统计 sheet 加"高项"列

### 前端改动

- `EntryPage.tsx:240-250` — 加 `<TableCell>高项</TableCell>` 列
- `SampleEntryPage.tsx` — 同样
- `StatsPage.tsx` — 同样
- 多个 records 列表页面（work / rd / sample_info）
