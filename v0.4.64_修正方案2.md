| ① 批量导入没下载模板 | ❌ v0.4.63 没做 | 实施跳过了方案的三步流程 |
| ② 高项是 Select 下拉不是自定义文本 | ❌ 实施错误 | 误解"自定义模式"为独立 CRUD |

其中 ④ 和 ⑤ 互相绑定：

```
用户删除 ICP 类型 → 后端 soft_delete(is_active=0) → audit_log 写入
       ↓
用户找不到已删类型在前端任何地方（回收站不显示）
       ↓
用户再创建 ICP 类型 → 后端 SELECT COUNT(*) where type_key='icp'（含 is_active=0）
       ↓
返回"类型标识『ICP』已存在"
       ↓
用户困惑："我删掉了为什么还说已存在？"
```

## 2. 修正 2：高项改为纯自定义文本

### 用户确认
"创建时是纯自定义文本" — 项目编辑时，高项是**文本输入框**（`<TextField>`），不是 Select 下拉，不是 Autocomplete。
"统计是优先填写关联的高项" — 统计时按项目关联的高项显示/分组。
"记录内支持修改高项" — 在记录详情里用户可以手动修改高项（覆盖项目默认值）。

### 技术方案

**数据库层**：

```sql
-- projects 表高项改为 TEXT
-- v0.4.63 加的 high_item_id 删除
-- v0.4.63 建的 high_items 表删除
ALTER TABLE projects ADD COLUMN high_item TEXT DEFAULT NULL;

-- work_records 加 high_item（创建时从项目复制，之后可独立修改）
ALTER TABLE work_records ADD COLUMN high_item TEXT DEFAULT NULL;
ALTER TABLE rd_work_records ADD COLUMN high_item TEXT DEFAULT NULL;
```

## 4. 修正 4+5：类型回收站 + 唯一性检查修正

### 4.1 后端唯一性 SQL 修正

**文件**：`src/repo/sample_info_type_repo.rs:58-59`

```diff
- "SELECT COUNT(*) FROM sample_info_types WHERE type_key=?1",
+ "SELECT COUNT(*) FROM sample_info_types WHERE type_key=?1 AND is_active=1",
```

这样软删除后用户再用同名 type_key 创建不会报错。

### 4.2 前端回收站重构（在 ManagePage.tsx）

原回收站：

```typescript
const [tr, setTr] = useState<WorkRecord[]>([]);
const loadTrash = async () => {
  const r = await getRecords({ include_deleted: true });  // 只查 work_records
  setTr(r.data.items.filter((x: any) => x.deleted_at != null));
};
```

改为：

```typescript
// 回收站子 Tab 状态
const [trashTab, setTrashTab] = useState<'records' | 'types' | 'projects'>('records');
// 每个 tab 独立状态
const [trRecords, setTrRecords] = useState<WorkRecord[]>([]);
const [trTypes, setTrTypes] = useState<SampleInfoType[]>([]);
const [trProjects, setTrProjects] = useState<Project[]>([]);

const loadTrash = async () => {
  const records = await getRecords({ include_deleted: true });  // work records
  if (records.code === 0) setTrRecords(records.data.items.filter(x => x.deleted_at));
  const types = await getSampleInfoTypesAll();                  // 含 is_active=0 的类型
  if (types.code === 0) setTrTypes(types.data.filter(t => !t.is_active));
  const projects = await getProjects({});                        // 含 is_active=0 的项目
  if (projects.code === 0) setTrProjects(projects.data.filter(p => !p.is_active));
};
```
