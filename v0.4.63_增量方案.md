## 1. 需求 1：新增"高项"关联

### 1.1 用户描述
"给项目新增一个关联信息，定义为'高项'，该关联为自定义模式，不在前台显示，仅用做统计时用，将'高项'作为新的记录列添加到所有相关的统计记录的列中"

### 1.2 现状分析

#### 现有的"关联"机制（以"关联检测方法"为例）

**数据库**：`projects` 表 + `project_method_links` 桥接表（M:N）
```sql

    project_id, method_id
)
```

**Project model**（`src/models/project.rs`）：


**前端截图 1** 显示了"编辑项目"对话框的"关联检测方法"段：按 method_type 分组（液相/气相/理化），每个 method 是一个 checkbox。

#### "高项"的语义分析

根据用户描述：
- **"自定义模式"**：暗示是 boolean flag（是/否），不是 M:N 多选
- **"不在前台显示"**：编辑项目时也不显示这个字段（用户不想被它打扰）
- **"仅用做统计时用"**：出现在统计 sheet 中作为列

**关键推断**：高项是一个**项目级 boolean 属性**。每个项目可以标记为"高项"或"非高项"。统计时按这个 flag 分组/筛选/汇总。

#### 实现推断

实际上有两种理解：
1. **简单 boolean**：`projects.is_high_item INTEGER DEFAULT 0`（"是"或"否"）
2. **多级分类**：`projects.high_item_level INTEGER`（高/中/低/无）

按用户描述"高项"（单一标签），应该是 **boolean**。但是应该在**管理页的"项目"Tab** 里有开关（虽然"不在前台显示"，但管理后台可以设置），不是完全隐藏。

### 2.3 实现建议

#### Excel 格式规范

第一行是表头，列定义：
| 列 | 必填 | 验证 | 示例 |
|---|---|---|---|
| A 用户名 | 是 | 非空、唯一 | `zhangsan` |
| B 密码 | 是 | 长度>=6 | `123456` |
| C 真实姓名 | 否 | | `张三` |
| D 所属部门 | 否 | 部门必须存在 | `研究部` |
| E 所属实验室 | 否 | 实验室必须存在 | `410` |
| F 角色 | 是 | 角色必须存在 | `普通用户` |
| G 启用 | 否 | 默认 1 | `1` |

**验证规则**：
- 用户名重复 → 跳过 + 报告
- 部门/实验室/角色不存在 → 跳过 + 报告
- Excel 解析失败 → 整批拒绝

**新 handler**：
```rust
async fn import_users(
    State((pool, _config)): State<(DbPool, Arc<AppConfig>)>,
    headers: HeaderMap,
    mut multipart: Multipart,  // 需要 .limit(50MB)
) -> Result<Json<ApiResponse<ImportResult>>> {
    require_admin(&extract_claims_from_headers(&headers)?)?;
    
    let mut file_data: Vec<u8> = Vec::new();
    while let Some(field) = multipart.next_field().await... {
        // 解析 multipart 拿 Excel bytes
    }
    
    // 1. 解析 Excel（用 crate calamine 或 rust_xlsxwriter 反向解析）
    let rows = parse_excel(&file_data)?;
    
    // 2. 验证每行 + 写库（事务）
    let mut result = ImportResult::default();
    for row in rows {
        match validate_and_insert(&pool, &row) {
            Ok(_) => result.created += 1,
            Err(e) => result.errors.push((row_index, e.to_string())),
        }
    }
    
    // 3. 写审计
    audit_repo::log("create", "users", None, "admin", &format!("批量导入 {} 个用户", result.created), "shared");
    
    Ok(Json(ApiResponse::ok(result)))
}

pub struct ImportResult {
    pub total: usize,
    pub created: usize,
    pub skipped: usize,
    pub errors: Vec<ExcelImportError>,  // {row: usize, message: String}
}
```

**关键依赖**：
- `calamine = "0.26"` — Excel 解析库（读 .xlsx）
- 或 `rust_xlsxwriter` 的反向能力（如果支持）
