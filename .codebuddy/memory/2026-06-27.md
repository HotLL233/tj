## v0.2.1: 导入功能独立模块化 + 即时反馈机制 ✅ (16:36)
- **核心目标**: 将 Rust-Tool Excel 解析逻辑封装为独立模块,通过 API 返回结构化结果,前端即时显示无需点击分类
- **后端模型** (`models/import_record.rs`):
  - 新增 `ImportResult` 结构体: success, total_rows_read, inserted, updated, skipped, sheet_name, columns_found, errors, warnings
  - 新增 `summary()` 方法生成人类可读摘要
- **Excel 解析器** (`utils/excel_reader.rs`):
  - 新增 `ParseResult` 结构体: records + sheet_name + columns_found + total_rows_read + skipped_rows
  - `process_sheet()` 改为返回 `Option<ParseResult>` 而非 `Option<Vec<ImportRecord>>`
  - 跟踪每行的跳过原因(日期/批号/数量无效),精确统计 skipped_rows
  - 收集检测到的列名(含列索引)
  - 错误信息包含 Sheet 名 + 前5行预览
- **数据库导入** (`db/import.rs`):
  - `batch_import()` 返回 `Result<(inserted, updated, errors)>` 三元组
  - 单条记录失败不中断整体事务,errors 收集所有失败原因
- **API 层** (`api/import_handler.rs`):
  - `POST /api/import/excel` 返回 `ApiResponse<ImportResult>` 结构化结果
  - message 字段使用 `result.summary()` 生成可读摘要
- **前端** (`StatsPage.tsx` + `types/index.ts`):
  - 新增 `ImportResult` TypeScript 接口
  - 新增 `importResult` / `importErr` / `importOpen` 独立状态
  - `hi()` 解析 ImportResult 并独立存储,不依赖 `er` 状态
  - 新增导入进度提示 Alert ("正在解析...")
  - 新增导入结果详情 Alert (新增N条/累加N条/跳过N行/检测列/工作表名) — **始终可见,无需点击分类**
  - 新增 Snackbar 底部弹出即时通知(4秒自动消失)
  - 错误信息也会独立展示,不依赖标签页
- **编译**: cargo build --release ✅ | tsc ✅ | vite build ✅
- **打包**: `installer\工作量统计工具_Rust_v0.2.1_Setup.exe` ✅
