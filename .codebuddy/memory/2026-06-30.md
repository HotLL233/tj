# 2026-06-30

## v0.2.10: 数据隔离 + 项目管理分层嵌套 ✅
### 核心修复
- **list()**: 默认 WHERE parent_id=0 仅返回顶级项目，避免显示全部方法
- **list_methods()**: 新增 WHERE parent_id>0 仅返回子级检测方法
- **include_methods=true**: 递归加载子方法到 ProjectResponse.methods[]
- **项目管理 tab**: 点击项目展开嵌套子方法列表（可编辑/删除）
- **方法管理 tab**: 使用 /api/methods 端点，仅显示子级方法
- **导入逻辑**: 保持不变（实验室→研发项目→检测方法三级）

### 新增 API
- GET /api/methods?method_type= → 子级检测方法列表
- GET /api/projects/:id/methods → 某项目的子方法
- GET /api/projects?include_methods=true → 含嵌套方法

### 前端改动
- Project 类型新增 parent_id + methods?: Project[]
- getMethods() / getProjectMethods() API
- 项目管理: 项目可点击展开，蓝色左边框缩进显示子方法
- 方法管理: 独立数据源(ms)，不与项目混合

## v0.2.11: 扁平导入(列间无从属) + 安装包同v0.2.7路径 ✅
### 核心简化
- 导入: 每行实验室+研发项目, 列2起每个非空单元格=独立方法, 无主从关系
- 方法名前缀推断类型: LC-→液相, GC-→气相, 其他→其他
- 存储: 扁平化, 每个方法独立projects记录, full_name存研发项目路径
- 管理页: 无嵌套展开, 所有项目/方法平铺显示, full_name显示研发项目来源
- 安装包: 统一输出到 `installer\工作量统计工具_Rust_v0.2.11_Setup.exe`

### 安装包路径
D:\桌面\工作量统计工具项目\installer\工作量统计工具_Rust_v0.2.11_Setup.exe
### 核心改动
- 导入解析: 实验室(A列) → 研发项目(B列) → 检测方法(C列起为类型列)
- 方法类型自动推断: LC-*→液相, GC-*→气相
- 表头映射: "液相方法"→液相, "气相方法"→气相, "理化方法"→理化, "ICP"→ICP, "热分析"→热分析, 其他→其他
- 数据结构: projects 表新增 parent_id 字段(研发项目=0, 检测方法=pid)
- 研发项目本身存为 project(method_type='研发项目', parent_id=0)
- 检测方法存为 project(parent_id=research_project_id)

### 文件变更
- models/project.rs: MethodImportItem 加 project_name/method_name, ImportSummary
- repo/project_repo.rs: batch_import_tree() 三级upsert, delete 级联删除
