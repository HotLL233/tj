### T02：后端方法导入智能识别

**文件**: `v0.3.3/src/repo/method_repo.rs` — `batch_import_column_split` 函数（修改）

**核心逻辑**:

现有 `import_mappings` 表已有通配规则 `('*方法*', 'methods', '其他', 90)`，会匹配含"方法"的列头并以类型"其他"路由到 methods 表。**v0.4.0 增强：在匹配后，从列头名提取真实类型**。

```
现有流程：
  import_mappings 匹配 '*方法*' → type="其他" → methods 表

v0.4.0 增强：
  1. import_mappings 匹配 '*方法*'
  2. 提取 type_name = header.replace("方法", "")  
     ┌─────────────────┬──────────────────┐
     │ 列头             │ 类型             │
     ├─────────────────┼──────────────────┤
     │ 液相方法         │ 液相             │
     │ 气相方法         │ 气相             │
     │ 离子色谱方法     │ 离子色谱         │
     │ 检测方法         │ 检测             │
     └─────────────────┴──────────────────┘
  3. 查 method_types 是否存在该类型
     ├─ 不存在 → INSERT INTO method_types (name, sort_order) VALUES (?, 10)
     └─ 存在   → 复用
  4. 逐行取方法名:
     ├─ 已存在于 methods 表 → 跳过（去重）
     └─ 不存在 → INSERT INTO methods (name)
  5. INSERT INTO method_type_links (method_id, method_type_id)
```

**关键边界处理**:
- 列头不含"方法"→ 走 import_mappings 原规则，不触发新逻辑
- 方法名已存在 → 直接跳过，不报错不覆盖
- 多个列产生相同类型名 → 复用同一 method_type 记录
- 仅在匹配 import_mappings 规则后才提取类型，不影响原有路由

### T03：EntryPage 信息全面化 + 移动端适配

**文件**: `project-root/frontend/src/pages/EntryPage.tsx`（修改）

**当前状态分析**:
| 字段 | 状态 | 问题 |
|------|------|------|
| 录入时间 | ✅ datetime-local input | — |
| 实验室名称 | ❌ | 页面完全不显示，仅通过 URL groupId 隐式确定 |
| 研发项目 | ⚠️ | 在每行副标题 "项目: XXX"，不够突出 |
| 录入人 | ✅ TextField | — |
| 方法 | ✅ | — |
| 类型 | ✅ Chip 标签 | — |
| 数量 | ✅ input | — |

**改进方案**:
1. **实验室名称**：页面标题区增加实验室名称显示
   - 调用现有 `getGroups()` API 获取全量分组
   - 本地 `find(g => g.id === gid)` → 取 name 显示在标题下方
2. **研发项目名**：每行从副标题升级到与类型标签并列，更突出
3. **移动端**：<600px 表单单列、按钮全宽、方法行卡片式堆叠

**移动端布局**:
```tsx
// 桌面端 (≥600px)
<Box sx={{ display: 'flex', gap: 2, flexWrap: 'wrap', alignItems: 'center' }}>
  <TextField label="用户名" sx={{ width: 140 }} />
  <TextField label="日期时间" type="datetime-local" sx={{ width: 200 }} />
</Box>

// 移动端 (<600px)
<Box sx={{ display: 'flex', flexDirection: 'column', gap: 2 }}>
  <TextField label="用户名" fullWidth />
  <TextField label="日期时间" type="datetime-local" fullWidth />
</Box>
```

**方法行移动端**:
```
桌面端: [方法名 | 系数 | 类型标签 | 项目名 | 数量输入 | 提交按钮] (flex row)
移动端: [方法名 + 类型标签 + 项目名]
        [数量输入 + 提交按钮] (flex column / stack)
```

## 七、共享知识

### CSS 约定
- 所有文本容器: `word-break: break-word`
- 表格外层容器: `overflow-x: auto`（移动端横向滚动）
- body: `overflow-x: hidden; max-width: 100vw`

### 响应式断点（MUI useMediaQuery）
- `xs`: `<600px` — 单列/全宽
- `sm`: `600–767px` — 两列
- `md`: `≥768px` — 多列/桌面

### 方法导入规则
- 仅匹配中文"方法"关键词的列头
- 类型名 = 列头.replace("方法", "")
- 已存在的方法名自动跳过（去重）
- 不存在的 method_type 自动创建（sort_order=10）
