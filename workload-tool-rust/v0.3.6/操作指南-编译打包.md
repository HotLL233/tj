### 第一步：编译

#### 选项 A - 使用 BAT 脚本

**双击运行**：`compile.bat`

#### 选项 B - 手动命令行

```cmd

Win + R → 输入 cmd → 回车

# 2. 进入项目目录
cd D:\桌面\工作量统计工具项目\workload-tool-rust\v0.3.5

# 3. 执行编译
cargo build --release

# 4. 等待编译完成（首次 5-10 分钟，增量 30-60 秒）
```

**编译完成标志**：
```

    Finished release [optimized] target(s) in XX.XXs
```

**验证编译产物**：
```cmd

```

应该看到文件大小约 15-20 MB。

## 🐛 常见问题排查

### 问题 1：`cargo: command not found`

**原因**：Rust 未安装或环境变量未配置

**解决**：
```cmd
# 检查 Rust 是否安装
rustc --version

# 如果未安装，访问 https://rustup.rs/ 安装
```

### 问题 2：编译报错 `linker 'link.exe' not found`

**原因**：缺少 Visual Studio C++ 构建工具

**解决**：
1. 下载安装 Visual Studio 2022 Community
2. 勾选"使用 C++ 的桌面开发"
3. 重新运行编译

### 问题 3：编译很慢/卡住

**原因**：网络问题或防火墙拦截

**解决**：
```cmd
# 配置国内镜像源
# 编辑文件：C:\Users\你的用户名\.cargo\config.toml
# 添加以下内容：

[source.crates-io]
replace-with = 'ustc'

[source.ustc]
registry = "https://mirrors.ustc.edu.cn/crates.io-index"
```

### 问题 4：编译成功但打包脚本报错

**原因**：PowerShell 执行策略限制

**解决**：
```powershell
# 以管理员身份运行 PowerShell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

# 然后重新运行打包脚本
.\build-and-pack.ps1
```

### 问题 5：`workload-tool.exe` 启动后立即退出

**原因**：可能缺少依赖或配置错误

**解决**：
```cmd
# 使用带控制台模式运行，查看错误信息
cargo run --features console
```
