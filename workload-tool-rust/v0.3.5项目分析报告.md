# 工作量统计工具 v0.3.5 项目分析报告

**分析时间**: 2026-07-02  
**项目版本**: v0.3.5  
**技术栈**: Rust + Axum + SQLite + React + TypeScript

---

## 一、项目概览

### 1.1 项目定位

这是一个**局域网多用户工作量录入与统计系统**的 Rust 重写版本，用于实验室环境下的工作量管理。

### 1.2 核心功能

- **多用户工作量录入**：支持实验室成员录入每日检测工作量
- **分组管理**：实验室分组（project_groups）管理
- **项目管理**：研发项目管理，支持多实验室、多方法关联
- **方法管理**：检测方法独立管理，支持系数配置和多类型标签
- **统计分析**：按用户、项目、类型、仪器等多维度统计
- **Excel 导出**：5 Sheet 导出（月汇总、每日、每周、原始记录、用户统计）
- **Excel 导入**：自动识别列头，批量导入工作量数据
- **样本记录**：独立的样本送检记录管理
- **审计日志**：完整的操作审计追踪
- **数据备份**：自动/手动备份，支持恢复和完整性校验

### 1.3 部署方式

- **单机部署**：Windows 系统托盘运行，无需安装
- **Web 访问**：局域网内通过浏览器访问（默认端口 8000）
- **数据存储**：SQLite 本地数据库（WAL 模式）

ProjectResponse {

**业务逻辑**:
- 支持多实验室关联（通过 `project_lab_links` 表）
- 支持多方法关联（通过 `project_method_links` 表）
- 删除前检查是否有 work_records 引用

### 3.2 方法管理模块（method）

**数据模型**:
```rust
MethodResponse {
    id, name, full_name,
    coefficient,  // 管理系数（用于统计加权）
    amount,       // 金额（v0.2.19+）
    notes, is_active,
    type_ids, type_names,  // 关联的检测类型
    created_at
}
```

**API 端点**:
- `GET /api/methods?type_id=`
- `POST /api/methods`
- `PUT /api/methods/:id`
- `DELETE /api/methods/:id`
- `POST /api/methods/import`（Excel 导入）
- `GET /api/method-types`（检测类型 CRUD）

**导入逻辑（v0.3.0 重构）**:
1. 读取 Excel 表头
2. 通过 `import_mappings` 表匹配列头模式
3. 根据匹配结果路由到不同目标表：
   - `*实验室*` → project_groups
   - `*研发*` / `*项目*` → projects
   - `*液相*` / `*气相*` 等 → methods + method_type_links
4. 批量插入数据，构建关联关系

### 3.3 记录管理模块（record）

**数据模型**:
```rust
RecordResponse {
    id, project_id, project_name, group_name,
    user_name, quantity, recorded_at,
    method_name,      // 关联的方法名（取第一个）
    method_type,      // 关联的方法类型（逗号分隔）
    created_at, deleted_at
}
```

**API 端点**:
- `GET /api/records?project_id=&group_id=&user_name=&start=&end=&page=&page_size=&include_deleted=`
- `POST /api/records`
- `PUT /api/records/:id`
- `DELETE /api/records/:id`（软删除）
- `POST /api/records/restore/:id`
- `DELETE /api/records/by-user/:user_name?start=&end=`

**Service 层验证**:
- `create`: 检查 quantity > 0，检查 project 存在
- `update`: 检查记录未被删除，检查是否有实际变更
- `delete`: 检查记录存在且未被删除

### 3.4 统计分析模块（stats）

**API 端点**:
- `GET /api/stats/summary?start=&end=&group_by=day|week|month&group_id=`
- `GET /api/stats/by-user`
- `GET /api/stats/by-project`
- `GET /api/stats/by-type`
- `GET /api/stats/by-instrument`

**系数计算逻辑（v0.2.17+ 适配）**:
```sql
-- 从 methods 表获取系数
SELECT COALESCE(SUM(wr.quantity * p.coefficient), 0.0) AS coefficient_score
FROM work_records wr
JOIN projects p ON wr.project_id = p.id
JOIN project_groups pg ON p.group_id = pg.id
WHERE wr.deleted_at IS NULL
```

**聚合维度**:
- 按时间：day / week / month
- 按用户：user_name
- 按项目：project_id + project_name + group_name
- 按类型：通过项目名称模式匹配（`%GC-%` 气相，`%LC-%` 液相）
- 按仪器：提取项目名称中的仪器代号

### 3.5 导出模块（export）

**导出格式**: Excel 5 个 Sheet

1. **月-汇总**：
   - 按实验室/项目树形结构展示
   - 液相/气相仪器列
   - 检测方法列
   - 月检测数量

2. **每日工作量**：
   - 按日期分组
   - 实验室/项目代号
   - 仪器、检测方法、数量

3. **每周工作量**：
   - 按周分组（Week1-Week5）
   - 每周数量动态列
   - 周汇总、液相/气相小计

4. **原始记录**：
   - 流水账格式
   - 字段：序号、日期、实验室、项目名称、仪器、检测方法、仪器类型、数量、录入人、管理分值

5. **用户统计**：
   - 按用户分组
   - 树形结构展示各实验室/项目工作量

**导出 API**:
```
GET /api/export/excel?start=2026-06-01&end=2026-06-30&group_id=1
```
