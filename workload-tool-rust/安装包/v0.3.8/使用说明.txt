# 工作量统计工具 v0.3.8 使用说明

## 快速启动

1. **启动服务器**
   - 双击 `启动.bat` 文件
   - 或直接运行 `workload-tool.exe`

2. **打开浏览器**

3. **停止服务器**

## 数据存储位置

所有数据存储在程序目录的 `data` 文件夹：

- `data/workload.db` - SQLite 数据库
- `data/backups/` - 数据库备份
- `app.log` - 运行日志

## 配置说明

如需修改配置，在程序目录创建 `config.toml` 文件：

```toml
server_port = 8000        # 服务器端口
db_dir = "data"           # 数据库目录
log_level = "info"        # 日志级别 (debug/info/warn/error)
```

## 导入功能说明（重要变更）

v0.3.8 采用**纯名称导入模式**：

1. 导入 Excel 后，项目/方法/实验室仅作为名称存储
2. **不会自动建立关联关系**（与 v0.3.6/v0.3.7 不同）
3. 导入的项目 `group_id = 0`，不挂靠任何实验室分组
4. 需要用户在管理页面手动编辑项目，选择关联的实验室和方法
5. 这样设计避免了自动关联的复杂性和不可靠性

### 导入后操作步骤
1. 导入 Excel 文件
2. 进入"项目管理"页面
3. 编辑导入的项目
4. 勾选要关联的实验室和方法
5. 保存

## 常见问题

### Q: 导入后看不到项目？
A: v0.3.8 导入的项目不会自动关联实验室。请在"项目管理"页面编辑项目，手动勾选关联的实验室。

### Q: 与 v0.3.6/v0.3.7 的区别？
A: v0.3.8 简化了导入逻辑，不再自动关联。导入后项目 `group_id = 0`，需手动选择关联。

### Q: 如何备份数据？
A: 复制 `data` 文件夹即可。程序每天自动备份到 `data/backups/`。

### Q: 端口 8000 被占用怎么办？
A: 创建 `config.toml` 文件，修改 `server_port` 为其他端口（如 8080）。

### Q: 如何升级版本？
A:
1. 关闭旧版本程序
2. 备份 `data` 文件夹
3. 替换 `workload-tool.exe` 文件
4. 复制 `static` 文件夹（如有更新）
5. 启动新版本

### Q: 如何验证导入功能？
A: 导入后检查：
- 审计日志包含时间戳（`audit_log` 表）
- 重名方法有多个类型关联（`method_type_links` 表）
- 项目 `group_id = 0`（`projects` 表）
