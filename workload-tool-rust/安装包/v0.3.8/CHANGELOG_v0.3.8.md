## 🎯 核心更新

### 纯名称导入模式（重大改进）

**v0.3.8 移除了项目-实验室自动关联功能**，导入后的项目/方法/实验室分组均为**纯名称存储**，不再自动关联到任何分组。

#### 设计理念
- ✅ **实验室分组**: 仅作为名称列表存在（`group_id` 独立管理）
- ✅ **研发项目**: `group_id = 0`，不挂靠任何实验室分组或伪分组
- ✅ **方法**: 纯名称 + 类型标签，不关联实验室
- ✅ **用户手动关联**: 导入后用户通过前端 UI 手动编辑项目，选择关联的实验室和方法

#### 与 v0.3.6/v0.3.7 对比

| 版本 | 项目 group_id | 自动关联 | 问题 |
|------|---------------|---------|------|
| v0.3.6 | "研发项目"伪分组 | ✓ 尝试自动关联（有缺陷）| 关联逻辑复杂且不可靠 |
| v0.3.7 | "研发项目"伪分组 | ✗ 删除了关联功能 | 仍挂靠伪分组，数据模型不清晰 |
| v0.3.8 | `0` (不关联) | ✗ 纯名称存储 | ✅ 清晰、简单、可靠 |

## 🐛 已修复漏洞（v0.3.6 遗留问题）

### 漏洞 #2: 方法重名冲突静默跳过 ✅ 已修复
**问题**: 导入时重名方法被静默跳过，类型关联不追加  
**修复**: 
- 重名方法不再跳过，而是追加类型关联
- 使用 `INSERT OR IGNORE` 确保类型不重复

```rust
// v0.3.8
let mid = if let Some(existing_id) = existing {
    // 已存在：仅追加类型关联（不覆盖已有数据）
    existing_id
} else {
    // 不存在：创建新方法
    tx.execute("INSERT INTO methods (name) VALUES (?1)", ...)?;
    method_count += 1;
    tx.last_insert_rowid()
};
```

---

### 漏洞 #5: 事务不完整 ✅ 已修复
**问题**: 导入无事务保护，失败时部分数据写入  
**修复**: 
- 添加 `unchecked_transaction()` 包装
- 所有 SQL 操作在事务内执行
- 失败时自动回滚，成功时统一提交

```rust
// v0.3.8
let tx = conn.unchecked_transaction()?;
// ... 所有插入逻辑 ...
tx.commit()?;
```

---

### 漏洞 #6: 类型名智能提取逻辑错误 ✅ 已修复
**问题**: `"方法列表"` 被提取为 `"列表"` 类型（垃圾类型）  
**修复**: 
- 白名单模式：仅提取已知类型
- 非白名单类型使用映射配置的 `default_type`

```rust
// v0.3.8
let known_types = ["液相", "气相", "理化", "ICP", "热分析", "质谱", "其他"];
let extracted = header.replace("方法", "").trim();
if known_types.contains(&extracted) {
    // 自动创建类型
} else {
    // 使用 default_type
}
```


### 漏洞 #9: 审计日志信息不完整 ✅ 已修复
**问题**: 导入审计日志缺少批次信息，无法溯源  
**修复**:

## 🐛 v0.3.8 补丁更新（2026-07-03 下午）

### 导出功能修复 ✅
**问题**: 导出的 Excel 文件只有 1KB（空数据）  
**原因**: 导出查询使用 `JOIN project_groups ON p.group_id = pg.id`，但 v0.3.8 项目的 `group_id = 0`，导致 JOIN 失败返回空结果集  
**修复**: 
- 将所有查询改为 `LEFT JOIN project_lab_links + LEFT JOIN project_groups`
- 通过关联表获取实验室信息
- 未关联实验室的显示为"未分组"

**影响范围**:
- Sheet 1: 各实验室项目方法对应表
- Sheet 2: 仪器-汇总
- Sheet 3: 项目-汇总
- Sheet 4: 实验室-汇总
- Sheet 5: 人员-汇总（原始记录）
- Sheet 7: 实验室总表

修复后的查询示例：
```sql
-- v0.3.8 补丁前（错误）
FROM projects p
JOIN project_groups pg ON p.group_id = pg.id  -- group_id=0 时失败

-- v0.3.8 补丁后（正确）
FROM projects p
LEFT JOIN project_lab_links pll ON p.id = pll.project_id
LEFT JOIN project_groups pg ON pll.group_id = pg.id
```
