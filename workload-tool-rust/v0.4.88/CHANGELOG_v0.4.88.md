# v0.4.88 更新说明

## 本次更新

- 新增研发项目状态字段：支持 `ongoing` / `archived`，归档项目会从前台 `active_only` 项目列表中隐藏，历史记录和关联数据保留。
- 项目管理页新增项目归档/重新启用操作；归档与停用分离，`is_active` 继续保留原停用/回收逻辑。
- 主数据导入模板改为简化结构：`主数据`、`项目关联`、`元数据`，并兼容旧版多工作表模板导入。
- `项目关联` 模板列调整为“关联实验室”和“关联方法”，关联方法支持多值分隔填写。
- 备份配置新增备份模式和同步目录字段；手动备份支持数据库备份与全量 zip 备份，全量包包含数据库快照和附件目录，并可复制到本地同步目录。

## 验证

- 已执行 `cargo test`，3 个后端测试通过。
- 已执行 `npm run build`，前端构建通过。
- 已执行 `cargo build --release`，release 可执行文件已生成。

## 注意

- v0.4.88 的高级备份 UI 先保持基础兼容；可通过配置接口保存 `mode=full` 和 `sync_dir` 后使用手动备份生成全量包。
- 全量 zip 恢复的后端辅助逻辑已预留，但本轮未在前端开放恢复入口。
