# v0.4.102 身份权限与会话安全版

发布日期：2026-07-20

## 身份与权限

- 用户增加永久用户 ID，登录仍使用用户名；鉴权、审计和记录创建人使用“用户 ID + 用户名”联合校验。
- 保留多角色分配和角色新增功能，多个角色的权限取并集。
- 内置系统管理员、分析检测员、分析检测组长、研发送样员、研发送样组长五个固定角色，固定角色不可修改、删除或调整权限。
- 系统管理员可新增、编辑、删除自定义角色并配置权限；分析检测组长仅可查看角色配置。
- 按角色落实分析检测、研发送样、样品登记、统计、回收站、审计和主数据管理的数据范围。
- 分析检测组长可创建和编辑分析检测员、研发送样员、研发送样组长，但不能创建或编辑系统管理员和同级组长。
- 用户 CSV 导入支持多角色，使用 `|`、`;` 或 `；` 分隔，并执行与手工创建相同的角色层级和实验室校验。
- 用户永久删除改为匿名化停用：保留用户 ID、历史记录和审计外键，撤销角色与会话并禁止再次登录。

## 会话安全

- 同一用户最多允许两台设备同时在线。
- 第三次登录自动取消最早的有效会话，保留最近两次登录。
- 会话删除后旧 Token 立即失效；旧设备下一次请求时自动清理本地登录并返回登录页。
- 每次登录使用独立会话标识，解决同一秒登录可能生成相同 Token 的问题。

## 方法、仪器与统计

- 方法名称允许重名，“方法名称 + 仪器”共同确定一个独立方法实例。
- 方法内部编号由系统自动生成并隐藏；仪器编号（如 LC-01）继续在前台显示。
- 仪器统计按仪器 ID 聚合，同名方法绑定不同仪器时分别统计检测量。
- 分析检测和研发送样记录保存创建人、方法、仪器等关键快照，主数据改名后历史记录仍可追溯。
- 统计和导出按稳定用户 ID 归集，用户改名不会拆分历史工作量；人员统计不应用单价倍率。
- 研发送样状态统一为“待取样 → 待检测 → 已检测”，分析检测员具备取样和完成检测权限。

## 数据范围与审计

- 分析检测统计：普通检测员仅查看和导出本人数据，分析检测组长和系统管理员可查看全部。
- 研发送样统计：普通送样员仅查看和导出本人数据，研发送样组长查看本实验室，分析组和系统管理员查看全部。
- 样品登记导出采用同一数据范围，并将日期筛选改为 SQL 参数绑定。
- 回收站统一展示分析检测、研发送样和样品登记记录，分别按本人、本实验室或全量权限恢复。
- 样品附件读取遵循记录可见范围；普通用户只能修改本人且尚未取样记录的附件。
- 审计、溯源、备份、系统设置、帮助文档和列配置接口改为实时读取当前角色权限，不再依赖旧 Token 中的管理员状态。
- 所有导出和主数据维护审计记录真实操作者用户 ID 与用户名。

## 数据约束与迁移

- 项目、实验室和仪器名称保持唯一；方法按“方法名称 + 仪器 ID”唯一。
- 升级发现历史重复主数据时不删除、不合并，记录迁移告警并暂缓创建对应唯一索引。
- 旧记录自动回填创建人、统计对象、实验室和仪器内部 ID；无法匹配的数据保留原文本快照。

## 主数据导入

- 模板版本更新为 v0.4.102，检测方法按“方法名称 + 对应仪器编号”填写。
- 项目关联支持同一项目多行填写实验室和方法，同名方法使用仪器编号区分。
- 示例表仅作填写参考，不参与实际导入。
- 模板下载、预检和正式导入统一校验主数据导入权限；导入审计记录实际用户 ID 和用户名。
