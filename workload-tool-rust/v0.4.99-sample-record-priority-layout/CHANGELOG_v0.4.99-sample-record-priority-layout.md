# v0.4.99 样品记录核心信息与附件优化版

## 记录信息层级

- PC 端列顺序调整为：状态、序号/编号、批号/数量、主要成分/检测元素/备注、实验室/项目、送样人/部门、时间/检测类型、附件、操作。
- 批号加粗并置于数量之前，主要成分加粗显示。
- 检测元素类自定义字段紧随主要成分显示，其他自定义字段仍显示在备注之后。
- 状态、序号、部门、实验室等短字段列宽压缩，释放核心信息空间。

## 附件显示

- 已有附件直接使用批量查询结果显示，不要求先点击附件数量。
- 附件名称取消单行省略号，改为自动换行显示完整文件名。
- 附件图标、文件名和删除按钮分列布局，多个附件纵向排列。
- 上传、打开、下载和删除逻辑保持不变。

## 手机端

- 手机卡片按批号、数量、主要成分、检测元素、备注、实验室/项目、送样人/部门、检测类型、时间、附件、操作顺序展示。
- 附件名称在卡片内完整换行，不产生页面级横向溢出。

## 版本隔离

- 从 `v0.4.98-rd-highitem-mobile-fix` 创建独立目录。
- 未复制旧版 `backend`、`dist`、`installer`、`target` 或 `node_modules`。
- 沿用原 AppId、应用名称和安装目录，支持覆盖安装并保留数据目录。

## 验证结果

- 后端测试：`18 passed, 0 failed`。
- 前端生产构建通过。
- `1920x1080`、`1366x768` 桌面视口无页面级横向溢出，九列按比例完整显示。
- `390x844` 手机视口自动切换为记录卡片，核心信息顺序正确且无页面级横向溢出。
- 长附件名称可直接显示并完整换行，无需先点击附件数量；页面控制台无报错。
