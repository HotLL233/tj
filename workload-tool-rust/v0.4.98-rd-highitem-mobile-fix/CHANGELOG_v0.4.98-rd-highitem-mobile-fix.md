# v0.4.98 研发送样高项与手机布局修复版

## 研发送样高项

- 修复研发送样记录查询漏掉 `high_item_snapshot` 的问题。
- 记录接口返回创建记录时保存的关联高项快照，项目后续修改不影响历史记录。
- 今日记录和独立研发送样记录页面均显示高项。
- 研发送样记录列管理新增“高项”内置列，默认显示、只读。

## 手机端录入

- 送样人和部门保持两列显示。
- 实验室、项目、检测类型和方法改为整行显示，避免选择值被窄列截断。
- 数量和批号保持两列，注意事项保持整行。
- 添加、删除、重置按钮在手机端等宽排列。

## 样品信息部门

- 修复样品信息记录响应未填充部门名称的问题。
- 根据记录的 `division_id` 查询并返回 `division_name`。
- PC 表格和手机卡片统一显示部门名称，不再显示数字 ID。
- 表单默认部门优先取用户绑定部门；未直接绑定时根据用户所属实验室补全。

## 版本隔离

- 从 `v0.4.97-mobile-entry-fix` 源码创建独立目录。
- 未复制旧版 `backend`、`dist`、`installer`、`target` 或 `node_modules`。
- 沿用原 AppId、应用名称和安装目录，支持覆盖旧版本并保留数据目录。

## 验证结果

- 前端 TypeScript 和 Vite 构建通过。
- Rust 测试 18 项全部通过，release 构建通过。
- API 实测研发送样记录返回创建时保存的高项快照。
- API 实测样品信息记录返回部门名称。
- 390px 手机视口无页面级横向溢出；实验室、项目、检测类型和方法宽度均为 324px。
- 研发送样记录页面实测显示高项列及高项内容。
- 样品信息手机卡片实测显示部门名称，不显示数字 ID。
- 正式 exe 使用 Windows GUI 子系统，Inno Setup 6.7.2 编译通过。
