# 分析检测统计卡片图表方案（v0.4.76 评审稿）

> 本文件仅为方案交付物。基准代码为 `workload-tool-rust/v0.4.75`，本次未修改任何产品源代码。

## 1. 目标

在保留现有统计口径、时间筛选、卡片跳转、明细表格和 Excel 导出的前提下，为统计页增加两级图表能力：

1. 顶部 6 个指标卡增加小型趋势/占比预览。
2. 点击任一顶部指标卡或下方统计入口卡后，在现有明细表上方展示该统计项的大图表。
3. 日期范围、快捷日期、检测人/实验室/项目/仪器/方法等现有筛选条件变化后，图表与表格使用同一批数据同步刷新。
4. 图表只做现有数据的可视化，不新增或改变统计公式。

## 2. 当前代码结论

- 页面入口：`frontend/src/pages/StatsPage.tsx`。
- 顶部指标卡：`frontend/src/components/StatsCards.tsx`。
- 当前点击行为：卡片通过 `setAc(tab)` 进入对应明细，返回后恢复入口卡网格。
- 日期变化后，现有 `useEffect` 会重新请求 summary、by-type、by-project、by-instrument；打开的明细也由 `ld2()` 重新请求。
- 已有统计接口：summary、by-user、by-project、by-type、by-instrument、by-division，以及 Sheet 1-11 预览接口。
- 当前依赖中没有图表库。
- `layout_stats` 已预留 `charts` 区域配置，但统计页目前没有真正渲染图表组件。

## 3. UI 结构

页面顺序保持为：

1. 页面标题。
2. 顶部 6 个指标卡，卡内增加不抢占主数字的小图表。
3. 原日期筛选与导出按钮。
4. 选中卡片后显示“图表 + 原明细表”；未选中时仍显示原统计入口卡网格。

桌面端大图表高度建议 300-340px；移动端 240px。图表右上角使用分段控件切换可用指标或图形，不改变数据源。顶部卡保持现有六列布局，移动端仍为两列。

## 4. 图表对应关系

| 卡片 | 默认图表 | 数据源 | 默认指标 |

| 总数量 | 折线图 | summary.details | total_quantity |

| 项目数 | 横向条形图 | by-project | total_quantity（按项目） |
| 实验室数 | 环形图 | by-project 前端按 group_name 汇总 | total_quantity（按实验室） |
| 方法类型 | 环形图 | by-type | total_quantity（沿用现有类型口径） |
| 按周统计 | 折线图 | summary(group_by=week) | 数量，可切记录数/管理分值 |
| 按月统计 | 折线图 | summary(group_by=month) | 数量，可切记录数/管理分值 |
| 检测人记录 | 横向条形图 | by-user | 数量，可切记录数/管理分值 |
| 事业部统计 | 组合柱状图 | by-division / Sheet 11 | 数量，可切记录数/管理分值 |

| 实验室-汇总 | 环形图 | Sheet 4 | 实验室数量，可切金额 |
| 检测人-汇总 | 柱状图 | Sheet 5 | 检测人数量 |

## 5. 刷新与状态规则

- 日期改变：立即进入加载态，卡片数值、小图、已打开大图和明细表使用同一筛选区间刷新。
- 类型/维度筛选改变：只更新当前图表与当前表格，不影响其他统计入口。
- 请求期间：保留旧图形轮廓并显示浅色 loading 遮罩，避免页面跳动。
- 无数据：显示坐标框和“当前条件下暂无数据”，数值为 0，不伪造占比。
- 请求失败：保留上一次成功结果，图表区显示错误提示和重试按钮。
- 连续快速改日期：取消或忽略过期请求，只有最后一次请求可写入页面状态。
- 图例过多：默认显示前 8 项，其余合并为“其他”；完整数据仍保留在原表格中。

## 6. 实现边界（后续获批后执行）

- 从 `v0.4.75` 复制建立独立 `v0.4.76` 目录，不修改旧版本。
- 推荐增加 Recharts 作为单一图表依赖，封装 `StatsChartPanel`、`MiniChart` 和数据适配函数。
- 先复用已有接口；只有在性能实测证明 Sheet 数据量过大时，才增加专门聚合接口。
- 图表数据适配使用纯函数，所有汇总字段和现有表格结果做一致性测试。
- 连接现有 `layout_stats.sections.charts.visible` 配置，使管理员可以控制图表区显隐。

## 7. 验收标准

1. 6 个顶部卡和全部统计入口均可打开对应图表。
2. 图表汇总值与同页表格/卡片数值一致。
3. 本周、本月、上月、近 7 天及自定义日期切换后自动刷新，无手动刷新按钮依赖。
4. 检测人、实验室、项目、仪器、方法筛选后图表和表格同步。
5. 原卡片点击、返回、编辑/删除记录、导出 Excel 行为无回归。
6. 0 条数据、1 条数据、超长名称、20+ 分类、接口失败均有稳定展示。
7. 桌面 1440px、平板 768px、手机 390px 下无重叠和横向溢出。

## 8. 方案风险

- 当前“方法类型”指标实际沿用现有 by-type 口径；若后续要改成方法管理中的真实类型，需要另立需求并明确历史数据兼容规则。
- Sheet 1-10 返回的是明细/导出预览数据，大数据区间可能较重。首版先按现有接口实施并设前端 Top N；性能不足时再增加只读聚合接口。
- 管理端可配置顶部卡片 key。后续实现需对未知 key 提供稳定兜底，避免自定义卡片点击后错误指向“按周统计”。
