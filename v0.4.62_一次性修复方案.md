## 二、当前仍然存在的真问题（4 个）

### Problem 1：PDF 上传提交后报"1 条失败"
**根因不确定**。v0.4.61 已删后端 MIME 检查，扩展名验证通过。
- 可能 1: 用户机器上装的还是 v0.4.60（旧 MIME 白名单拒绝 PDF）
- 可能 2: axum Multipart 解析某些 PDF 时 `next_field()` 报错
- 可能 3: 记录刚创建后外键约束检查与 WAL 同步时序问题

**需要你帮我确认**：打开浏览器 DevTools → Network → 找 `POST .../attachments` 请求 → 截 Response body 给我

### Problem 2：提交后记录内看不到附件
**根因**：`attachmentsByRow` 的 useEffect 在 submit 后 `load()` 时触发。`load()` 调 `getSampleInfoRecords` → setRecords → 触发 `useEffect([records])` → `batchGetSampleInfoAttachments`。

**但 uploadAttachment 也是 await 顺序执行的——upload 完才 load()，所以时序应该正确**。除非 upload 本身失败了（P1 的 catch 分支）。

**如果 upload 成功了但显示不出来**：查看 batch API 返回数据。

### Problem 3：新建类型"已存在"但列表没有
**根因已定**：v0.4.59 的 create 函数在 INSERT commit 后 init_for_type 才执行，init 失败返回 Internal，**但类型已在 DB 中**。
你是第一次试了没成，第二次再试报"已存在"——DB 里有垃圾行。

**不修代码**，只清数据：
```sql
DELETE FROM sample_info_types WHERE type_key = '质谱';
```

### Problem 4：BackToTop 按钮"消失"
**代码在、引用在、MD3 没变**（v0.4.59/60/61 完全一致）。
根因是：`<Zoom in={visible}>` 只在 `window.scrollY > 200` 时显示。管理页内容短，没滚到 200px。

**不修代码**，只是用户还不知道它怎么工作。滚长列表时自动出现。

## 三、修复策略

### 策略一：建立"一次复现再修"的流程（最重要）

| 角色 | 做什么 |
|------|--------|
| **你** | 打开 DevTools Network 标签，做一次"选 PDF → 提交登记"的完整操作 |
| **你** | 截 Network 里 `attachments` 请求的 Response body 图 |
| **我** | 拿到图后：把 test.db 复制到沙箱 → `cargo run` 启动 → 复现完整步骤 → 看到真实错误 |
| **我** | 定位根因，一次性修 |
| **我** | 用同一张 test.db 再跑一遍 → 确认修复生效 |
| **我** | 打包前写验证报告（不是 CHANGELOG，是验证报告） |

### 策略二：如果没有真实 Network 数据

那我就做**全路径检查**——把附件上传的完整链路全部走一遍，不管有没有报错，测这些：

```
1. 用 curl 直接 POST multipart 到 /api/sample-info/1/attachments 传 PDF
2. 看后端 console 有没有错误日志
3. 查 sample_info_attachments 表有没有记录
4. 查物理文件 data/attachments/ 有没有文件
5. 如果 1-4 都 OK，说明后端没问题，问题在前端 axios/FormData
```

然后汇报每一步的结果给你，你看到底哪里断了。
