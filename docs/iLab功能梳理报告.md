## 1. 产品定位

| 维度 | 结论（基于真实抓取） |
|------|------|
| 产品名 | **iLabPlus 智慧实验室**（前身"Mr.F 试剂耗材管家"，宣传语"做最好的耗材试剂管理软件，永久免费"） |
| 形态 | **云端 SaaS** 实验室智能化管理平台（网页版 + 电脑/Pad/手机多端） |
| 服务对象 | 科研实验室：高校、研究所、医院课题组等；免费版面向 ≤10 人中小型团队，专业版覆盖 ≤100 人中大型团队/企业部门 |
| 核心定位 | 实验室**试剂耗材 / 库存 / 采购 / 仪器 / 经费**的一体化运营与资产管理平台 |
| 核心卖点 | ① 永久免费基础版；② 试剂耗材极速查询（号称十万级 2 秒）；③ 流向跟踪与低库存/近效期预警；④ 仪器在线预约与共享；⑤ 多角色协作；⑥ 多维度统计；⑦ 数据安全（防火墙/加密/定时备份） |
| 商业模式 | 免费版（0 元）→ 经济版 2099 元/年（40 人）→ 专业版 4999 元/年（100 人），付费差异主要在人数上限、仪器高级统计、文档云空间、询价/合同管理 |
| 用户规模 | 官网宣称"来自 6000 所实验室用户的真实感受" |

> ⚠️ **关键边界判断**：iLabPlus 是"实验室运营/资产/耗材"管理平台，**并非完整意义的检测型 LIMS**。
> 其帮助文档与官网**未出现**样本/送样管理、检测任务/工单、方法/标准管理、检测记录与报告自动生成、
> 合规审计（GLP / 21 CFR Part 11 / 电子签名）等典型 LIMS 能力。若目标是"功能等效 LIMS"，需先明确：
> 是等效 iLabPlus 的**运营管理能力**，还是在此外补上**检测闭环**形成真正 LIMS（见第 5、6 节）。

## 3. 关键业务流程

**① 采购闭环（核心主干）**
```
成员申购 → 申购审批(若开启) → 采购员建订单(可并单/从申购生成)
  → 比价/询价(可选) → 采购审批(按金额阈值) → 发送供应商(短信/邮件)
  → 在线交流确认 → 到货订单入库(批量) → 库存增加
  → 经费: 新增支出+上传发票 → 批量标记报销
```

**② 库存闭环**
```
建库存(手动/导入) → 分类/标签 → 领用(审批) → 统一出库
  → 历史全链路追踪 → 低库存/近效期预警 → 触发补货申购(回到①)
```

**③ 仪器闭环**
```
添加仪器(档案+照片/二维码) → 设开放时间 → 成员预约 → 预约审批(若开启)
  → 使用 → 保养记录 → 使用率/空置率统计(付费)
```

**④ 审批流特征**
- 三类独立规则：领用 / 申购 / 采购，均可按"发起人+对象+审批人"配置。
- 采购审批支持**邮件异地审批，无需登录系统**（依赖邮件通道）。

## 4. 能力边界与特色

**差异化特色（iLab 强调的卖点）**
- 永久免费基础版，降低中小实验室使用门槛。
- 试剂耗材**极速查询**（十万级 2 秒）与**流向跟踪**。
- 低库存 / 近效期**主动预警**，减少浪费。
- 仪器**在线预约 + 共享 + 使用率/空置率统计**（付费）。
- 多端（电脑/Pad/手机）+ 微信管理，协作便捷。
- 采购与供应商深度打通：**在线商城、在线招商、比价询价、在线交流、短信通知**。

**能力边界（明确缺失，对"等效 LIMS"至关重要）**
| 缺失能力 | 说明 |
|----------|------|
| 样本 / 送样管理 | 无样本登记、条码、流转追踪 |
| 检测任务 / 工单 | 无检测任务下发、排程、状态机 |
| 方法 / 标准管理 | 无检测方法、SOP、标准库 |
| 检测记录与报告 | 无原始记录录入、无报告自动生成/模板 |
| 合规与审计 | 无 GLP、21 CFR Part 11、电子签名、审计追踪的明确能力 |
| 高级 BI 看板 | 仅有基础多维统计报表，非实时可视化大屏 |
| 多实验室 | 商业页标注"多实验室管理（开发中）" |
| 本地/私有部署 | 纯云端 SaaS，无离线/私有化选项 |

## 5. 本地化改造要点

> 目标：功能等效但**私有部署、离线运行**。以下按优先级与"云依赖替换"分类。

### 5.1 必须有的核心（P0 — 构成运营闭环的底座）
- 库存管理：分类 / 标签 / 批次 / 预警 / 领用 / 统一出库 / 历史追溯 / 导出
- 采购管理：申购 / 订单 / 审批 / 入库
- 经费管理：经费本 / 预算 / 支出 / 发票 / 报销
- 仪器管理：档案 / 预约 / 审批 / 保养 / 记录
- 供应商管理：**改为本地供应商目录**（移除在线商城/招商）
- 审批流引擎：领用 / 申购 / 采购 三类规则化配置
- RBAC 权限与角色（实验员/库管/采购/主管/仪器管理员/系统管理员）
- 基础统计与报表导出
- **本地数据库 + 离线运行 + 本地文件存储**（替换云端）

### 5.2 锦上添花、可后做（P1/P2）
- 仪器共享 / 空置率 / 使用率统计（高级分析）
- 文档管理（以本地文件服务器 / 对象存储如 MinIO 替代云空间）
- 多实验室管理
- 订单合同管理
- 比价 / 询价（改为本地供应商横向比价）

### 5.3 依赖云端 / 外部服务（需替换为本地方案）
| 原云依赖 | 本地化替换方案 |
|----------|----------------|
| 短信网关（订单/留言通知供应商） | 站内通知 + 本地 SMTP 邮件；如需短信可后接企业自有 HTTP 短信网关 |
| 云存储（文档管理 500MB/1GB） | 本地磁盘 / MinIO / NAS |
| 在线商城 / 供应商网络 / 在线招商 | 移除或仅保留本地供应商主数据 |
| 微信集成 | 本地移动端（PWA / App）替代 |
| 协助导入（平台人工整理） | 自助 Excel 导入 + 前端校验 + 模板下载 |
| 邮件异地审批 | 系统内审批（登录后审批） |
| 积分商城（gamification） | 可移除，非业务核心 |
| 服务器定时备份 / 加密 | 本地备份策略（定时快照 / 异地备份脚本） |

## 6. 结论与给架构师的建议

1. **iLabPlus 是"实验室运营/耗材/资产"管理平台，不是完整检测型 LIMS。** 其能力集中在
   库存、采购、经费、仪器预约、供应商、审批、RBAC、基础统计与协作，且大量能力依赖云端与供应商生态。
2. **"功能等效 LIMS"需先定基线**：若以 iLabPlus 为基线，则第 5.1 节的 P0 模块即为最小可行产品（MVP）；
   若目标是真正 LIMS，则必须在 P0 之外**补建样本管理、检测任务/工单、方法标准、检测记录、报告生成、合规审计**——
   这些 iLabPlus 完全不具备，属于增量而非等效。
3. **云依赖可系统性替换**：短信、云存储、在线商城、微信、人工导入、邮件审批均有明确的本地替代路径，
   离线私有部署在技术可行，但"供应商网络/在线招商/比价"这类依赖平台双边网络的特性无法本地复现，应裁剪。
