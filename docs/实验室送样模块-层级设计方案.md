# 实验室送样模块 — 层级设计方案（修订版）

> **核心理念**：送样模块 = 纯录入日志，无独立统计。统计分类在工作量统计界面中完成。
> **一级导航**：首页分为"实验室送样入口"和"工作量录入入口"，进入不同的下级层级。

## 三、数据模型

### 3.1 数据库表（`migrations.rs` 追加）

```sql
-- 送样记录表（一本送样单）
CREATE TABLE IF NOT EXISTS sample_records (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    project_id    INTEGER NOT NULL REFERENCES projects(id),
    group_id      INTEGER NOT NULL REFERENCES project_groups(id),
    user_name     TEXT    NOT NULL,                      -- 送样人
    sample_name   TEXT    NOT NULL,                      -- 样品名称/编号
    sample_count  INTEGER NOT NULL DEFAULT 1 CHECK(sample_count > 0),
    unit          TEXT    NOT NULL DEFAULT '个',          -- 单位(个/支/瓶/批)
    batch_no      TEXT    DEFAULT '',                    -- 批次号(可选)
    notes         TEXT    DEFAULT '',
    submitted_at  TEXT    NOT NULL,                      -- 送样时间
    created_at    TEXT    NOT NULL DEFAULT (datetime('now','localtime')),
    deleted_at    TEXT                                   -- 软删除
);

CREATE INDEX IF NOT EXISTS idx_sample_project ON sample_records(project_id);
CREATE INDEX IF NOT EXISTS idx_sample_group   ON sample_records(group_id);
CREATE INDEX IF NOT EXISTS idx_sample_user    ON sample_records(user_name);
CREATE INDEX IF NOT EXISTS idx_sample_date    ON sample_records(submitted_at);
CREATE INDEX IF NOT EXISTS idx_sample_deleted ON sample_records(deleted_at);
```

> **简化说明**：去掉批次表(原 `sample_batches`)，简化为一张 `sample_records` 表。每行 = 一次送样记录(什么人/什么时候/送了什么项目的什么样品/多少)。

### 3.2 实体关系

```
project_groups
  ├── projects
  │     ├── work_records  (现有检测记录)
  │     └── sample_records (新增送样记录)
  │
  ├── sample_records.group_id  ← 冗余 group_id 便于按实验室筛选
```

## 四、API 路由（仅 CRUD，无统计）

### 4.1 路由注册（`src/api/sample_handler.rs`）

```rust
pub fn router(pool: DbPool) -> Router {
    Router::new()
        .route("/api/samples",          get(list).post(create))
        .route("/api/samples/:id",      get(get_one).put(update).delete(soft_delete))
        .route("/api/samples/restore/:id", post(restore))
        .with_state(pool)
}
```

## 九、关键设计决策

| 决策 | 理由 |
|------|------|
| **单表 `sample_records`** | 送样只需要记录"谁送了什么的多少"，不需要批次+明细双表复杂性 |
| **冗余 `group_id`** | 方便按实验室筛选送样记录，避免 JOIN 查询 |
| **无独立统计** | 统计数据汇集到现有工作量统计界面中，按 `source` 分类 |
| **首页双入口** | "实验室送样"和"工作量录入"并列，用户按任务类型选择 |
| **复用 `projects` 表** | 项目方法库共享，送样和检测引用同一组方法 |
| **软删除** | 对齐现有架构，送样记录可恢复 |
